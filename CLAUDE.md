# CLAUDE.md — 作業ルール（必読）

このリポジトリで Claude Code が作業する際の必須ルール。作業前に必ず確認すること。

## 0. 作業ログ（必須）

- 何か変更したら、`docs/WORKLOG.md` の先頭（新しい順）に「日付・何を・なぜ・関連ファイル・ユーザー側の作業」を追記する。
- コードに残らない設定値（各種ID・トークン・外部サービス設定）は必ず WORKLOG か `docs/` に控える。
- 「あのとき何をやったか」を聞かれたら、まず `docs/WORKLOG.md` を読む。

## 1. 操作・実行の方針（最重要）

- **ユーザーに PowerShell / コマンド操作を依頼しない。** `git`・`npm`・`build`・`lint`・`commit`・`push` などは
  すべて Claude 側で実行する。「このコマンドを実行してください」という依頼は禁止。
- 危険な操作（破壊的変更・force push 等）のみ事前確認する。通常作業はそのまま進める。
- 既存ファイルを破壊しない。構成が異なる場合は全置換せず、現状に合わせて整理する。

## 2. サイトの性格（絶対に守る）

- サイト名は **城北セレモニーサポートセンター**。**運営・施行：川口典礼** を必ず明記する。
- 中立サイト・公共機関・公式施設サイトのように見せない。
- 全ページで「本サイトは戸田斎場・舟渡斎場・北区セレモニーホール・蓮根レインボーホール等の
  **公式施設サイトではありません。運営・施行は川口典礼です。**」の趣旨を必要箇所に明記する。
- 川口市・めぐりの森・川口メモリアルホールを主キーワードにしない（本体サイトの役割）。

## 3. 表現禁止ルール

次の表現は使わない：
最安／激安／必ず／絶対／追加費用なし／No.1／どこよりも安い／公式／公認／
北区民のみ／板橋区民のみ／競合批判／不安を過度に煽る表現。

価格を出すときは必ず以下を併記する：
- 「目安」「税込」
- 「内容により変動する」
- 「正確な費用は個別にお見積り」
- 式場料金・火葬料金・宗教者へのお礼・返礼品・飲食費などは内容により変動する旨

## 4. SEO / AIO 方針

- 各ページに title / description / canonical / OG / Twitter / robots / JSON-LD を設定。
- JSON-LD は WebSite / FuneralHome（LocalBusiness）/ BreadcrumbList / FAQPage。
- 存在しない住所・営業所を作らない。北区・板橋区に実店舗があるように見せない（`areaServed` で地域性を表現）。
- title・H1 をページ間で重複させない。薄い量産ページを作らない。
- 詳細は `docs/SEO_AIO_GUIDE.md`。

## 5. デザイン方針

- 紺基調・白／生成りで落ち着き・金をアクセント。上品で信頼感のあるトーン。
- スマホ最優先。モバイル固定CTAは2つ（電話／相談・見積り）、電話を最優先。
- 派手な安売り・煽り・公共機関風・ポータル風の価格訴求は避ける。
- 詳細は `docs/DESIGN_SYSTEM.md`。

## 6. 実装ルール

- 文言・電話番号は `app/config/site.ts` で一元管理する（直書きしない）。
- データは `data/`（halls / plans / areas / faqs）で管理し、後から編集しやすくする。
- `@/` エイリアスを使用（`tsconfig.json` 設定済み）。import の大文字小文字に注意（Linux/Vercel で落ちる）。
- 存在しない画像参照でビルドエラーを出さない。
- `node_modules` / `.next` / `.vercel` は git 管理しない。

## 7. commit / build / push 方針

作業の最後に必ず順に実行する：
1. `git status`
2. `npm install`
3. `npm run build`
4. `npm run lint`（あれば）
5. エラーがあれば修正 → 再度 `npm run build`
6. `git status`
7. `git commit`
8. `git push origin main`

- TypeScript エラー・build エラーを残さない。
- commit メッセージは変更内容が分かる日本語または英語で簡潔に。
