## 4. メタデータ方針

- 各ページに title / description / canonical / OG / Twitter / robots を設定（`lib/seo.ts` の `buildMetadata`）。
- title・H1 はページ間で重複させない。description はページ固有・各150字前後。
- canonical は本番ドメイン＋末尾スラッシュ（`next.config.ts` の `trailingSlash: true`）。
- `app/sitemap.ts` / `app/robots.ts` を自動生成。

## 5. JSON-LD 方針（`lib/jsonld.ts`）

- **WebSite**：全ページ（ルートレイアウト）。
- **FuneralHome（LocalBusiness 系）**：全ページ。`areaServed` に北区・板橋区。
  - 存在しない住所・営業所は記載しない。`parentOrganization` に川口典礼。
- **BreadcrumbList**：各下層ページ。
- **FAQPage**：`/faq/` と各斎場ページのFAQ。

## 6. FAQ設計（AIO）

- 質問は実際の不安・検索意図に対応（利用可否・形式可否・搬送・空き確認・見積り・運営主体・公式かどうか）。
- 回答は簡潔かつ自己完結。必要箇所で「公式施設サイトではない／運営・施行は川口典礼」を明記。
- 全体FAQは `data/faqs.ts`、斎場固有FAQは `data/halls.ts` の `faqs`。

## 7. 内部リンク設計

- トップ → 各エリア・各斎場・各プラン・FAQ・相談ページ。
- 斎場一覧 → 各斎場詳細・各プラン。
- 各斎場 → 各プラン・エリア（北区/板橋区）・相談ページ。
- 各プラン → 戸田斎場ほか各斎場・エリア・相談ページ。
- リンクは検索意図の近い順に配置し、相互に行き来できるようにする。

## 8. カニバリ回避方針

- 地域ページ（エリア）と斎場ページは「地域軸」と「施設軸」で役割を分ける。
- プランページは「形式軸」の横断受け皿とし、地域・施設名はリンクで補完（重複H1を作らない）。
- 川口市軸のKWは本サイトで作らない。本体サイトと食い合わせない。
